Magma users should run the SetMemoryLimit() function to avoid memory leaks inconveniencing others. The simplest way is to put the line SetMemoryLimit(n); in your Magma startup file, where n is about half the RAM of the machine in bytes: for example, on fermat use SetMemoryLimit(32*10^9);. In fact, it might be better to use a much smaller value, say 1/10 of the above, as your default, since it seems that most of the time when the memory runs away it's because of some fault or bug. You can always increase the memory for a particular run by putting a call to SetMemoryLimit in your script. You can see the current value with GetMemoryLimit(); where 0 means no limit and is the default.
